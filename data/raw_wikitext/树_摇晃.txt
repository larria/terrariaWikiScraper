+++ Silver Coin drop rates, Desktop 1.4.0.5 +++

From the source code, we can easily get the chances for "1-20sc", "2-40sc", and "3-60sc":
P(1-20)=29/600; 	P(2-40)=29/720; 	P(3-60)=29/3600.

However, this is not very useful for determining the drop chance of "14sc", for instance, because "14sc" is encompassed by all three of those chances.

Therefore, we need to get P(1), P(2), P(3), P(4-20), P(21-40), and P(41-60).

In order to do that, we express the chances above as P_a, P_b, and P_c, respectively, so we can work with them:
P(1-20)=29/600=P_a 	// encompasses the coin amounts from "1sc" to "20sc", all of which have an equal chance
P(2-40)=29/720=P_b 	// encompasses the coin amounts from "2sc" to "40sc", all of which have an equal chance
P(3-60)=29/3600=P_c 	// encompasses the coin amounts from "3sc" to "60sc", all of which have an equal chance


注：
To get P for one coin amount from the values above, we multiply it by (1/<number of values>): P_a(1)=P_a*(1/20)=29/12000.
We can safely say that P_a(1)=P_a(2)=...=P_a(20), P_b(2)=P_b(3)=..., or P_x(m)=P_x(n) in general, because all have the same chance. This also implies P_x(m-n)=P_x(o-p) where m-n=o-p (actual minuses in this last equation, not range hyphens), e.g. P_a(1-5)=P_a(2-6)=...=P_a(16-20). We will use this in the calculations below.
We cannot say P_a(10)=P_b(10)=P_c(10), however, because P_a encompasses 20 values, P_b encompasses 39 values, and P_c encompasses 58 values.


Start calculations:
