{{formatting subpage}} It is furthermore used to print each table row on [[Mobile version history]].

It takes multiple parameters:

{| class="terraria"
! Name !! Description
|-
| {{code|anchor}} || Name of the anchor for this table row. Defaults to the value of the {{code|$v}} parameter, so it is unnecessary in most cases.
|-
| {{code|v}} || Name of the update, i.e. version number.
|-
| {{code|platform}} || The console/mobile platforms on which the update was released, separate multiple with a slash ({{code|/}}). See [[Template:Icon#All valid icons|Template:Icon]] for all recognized platform names.
|-
| {{code|name}} || "Trivial name" of the update, e.g. "Hotfixes". Will be displayed in small gray text.
|-
| {{code|date}} || Date of the release, in the format {{code|YYYY/MM/DD}}, e.g. "{{#time:Y/m/d}}".
|-
| {{code|ref}} || Link to a source for the release date, e.g. via {{tl|code=y|forums thread}}.
|-
| {{code|rawref}} || Custom wikitext for the reference, will not be wrapped in a <code>&lt;ref&gt;</code> tag like {{code|$ref}}. Can be used, for instance, if there are multiple references. Takes precedence over {{code|$ref}}, should both be specified.
|-
| {{code|note}} || Text that will be displayed in italics at the top of the "Changes" cell.
|-
| {{code|changes}} || Details about the update. Should be wrapped in {{tl|code=y|collapse top}}/{{tl|code=y|collapse bottom}}, unless there are only a few sentences.
|}

It is possible that an update was released under different names or on different dates on the different platforms. To reflect this, append the platform name(s) to the {{code|$v}}, {{code|$date}}, {{code|$ref}}, and {{code|$rawref}} parameters with a hyphen, i.e. <code>$v-&lt;platform&gt;</code>, <code>$date-&lt;platform&gt;</code>, etc. The platform names must be valid as per [[Template:Icon#All valid icons|Template:Icon]], and multiple are to be separated with another hyphen. If the non-platform-specific {{code|$v}} or {{code|$date}} parameters are specified, then the respective platform-specific ones have no effect.

For instance, if the update "1.234" was released on Jan 1, 2018 on Xbox One and PlayStation 4, but on Jan 15 on Switch, the following code could be used:
<pre>
| v = 1.234
| date-ps4-xbox one = 2018/01/01
| date-switch = 2018/01/15
| ref-ps4-xbox one = {{forums thread|some-thread-about-xbox-and-playstation-release}}
| rawref-switch = <ref>{{forums thread|some-other-thread-about-switch-release}}</ref><ref>Some external reference about Switch release</ref>
</pre>
